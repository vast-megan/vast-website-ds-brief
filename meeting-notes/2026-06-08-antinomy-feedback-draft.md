# Antinomy Feedback — June 8, 2026
*Sent by Megan to Antinomy on June 8, 2026*

---

**Overall**
Really strong direction — the white space, type scale, and overall hierarchy are a big improvement. The "quiet until loud" concept resonates well with how we think about the brand. Here's our section-by-section feedback:

**Navigation**
Positive direction overall — we like how it scales and how the frosted underlay transitions on lighter backgrounds. Please switch the font from Inter to Owners, and use square corners instead of rounded for now. We'd love to see what the hamburger menu would actually reveal before we fully commit to it. One thing to explore: could the primary product items (Haven Station, Satellites, Lunar, Missions) be centered, with the hamburger offset to the right side?

**CTAs / Buttons**
The CTA buttons on the vapor grey section are reading as secondary because the background is too light. A possible fix: use frosted or moonrock warm-gray buttons on vapor / warm gray backgrounds. **Decision: go all-square corners on all buttons for now** — we'll define any rounded-corner moments later once we see the full component set.

**Announcements module**
The current treatment splits the headline/type moment and the media into two separate sections — we'd like to see them paired together. We need two distinct components here:

1. **Featured announcements (top of section):** Up to 3 key announcements, treated as high-importance, prominent content. To test: use the France announcement, the Czech announcement, and the latest Satellites announcement as the 3 featured items.
2. **Latest updates (further down the page):** A smaller card grid or slider showing up to 6 updates — these are separate from the 3 featured announcements above and should not duplicate them.

**Type / Eyebrows**
Use Phonic Mono for all eyebrow/label text — it's missing from the prototype and it overall needs to be larger. Getting the eyebrow size right is really important and worth spending time on.

**Orange CTAs**
The orange buttons are too heavy and also don't pass accessibility contrast (orange doesn't work on white or black). We want to scale them back. Our thinking: orange should show up only in very intentional moments — potentially as a hover state (unexpected, quiet-until-loud) rather than a persistent fill. We'll define the exact rule, but for now please default to white/neutral buttons.

**Button icons**
Please use this rule going forward: **plus (+)** = on-page interaction (user stays on the page), **arrow (→)** = navigating away to another page. Footer links don't need any icon — they're just links. The caret removal is fine as long as the arrow rule is applied consistently.

**Haven Station section / Product phases**
As you're designing the product feature sections, we want to make sure we're thinking about how products appear at different stages of investment. Not every product will have WebGL assets right away — some may only have a PNG at launch. We need these to feel like the same component at different phases, for example:

- Phase 1 = static PNG
- Phase 3 = WebGL

Haven Station and Lunar may appear on the same page, but Lunar may be earlier in its investment cycle. How can they feel like parallel, equally intentional product moments even when the asset complexity is very different? Please think through what this spectrum looks like and how the component handles each phase.

**Big scroll / type moments**
We love the impactfulness of the large type moments. We understand some of this might just be prototype limitations, but we want to remember to keep the user moving through the page quickly and not get caught in slow/fixed scrolls.
